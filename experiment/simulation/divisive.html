Instructions

  1. Select the number of nodes for your graph and click on the SUBMIT button.
    1. If nodes equal 4, follow the below steps:

      Create the complete graph by adding edges between every pair of nodes.

      1. Enter the values for the source node, destination node, and the weight/cost of the edge. Click on the SUBMIT button.
      2. Repeat step 3 until you have added edges between every pair of nodes.
      3. Finally, click on the NEXT button to initiate the computation phase for the divisive algorithm.
      4. Now, jump to step 2.
    2. If nodes equal 5, follow the below steps:
      1. Click on the START button.
      2. Now, click on the highlighted cell of the table (EDGE-COST) and enter the required value in the pop-up box.
      3. Repeat the above step for every cell of the table.
      4. Now, jump to step 2.
  2. Compute the Minimum Spanning Tree (MST) using Kruskal's algo. for the given adjacency matrix by clicking on the DRAW MST button.
  3. Enter the edge name from the sorted list of edge For example, 1-2 and then repeat this step until the MST is created.
  4. Progress to the next step by clicking on the NEXT button to identify "Singleton Clusters".
  5. Form a new cluster by breaking the link associated with the highest cost. Click on the BREAK button to proceed.
  6. Click on the NEXT button to navigate to the CLUSTERS page.
NOTE: Read the alerts carefully.

DEFINITION

Complete Graph:

A simple graph with n vertices is called a complete graph if the degree of each vertex is n-1, that is, one vertex is attached with n-1 edges or the rest of the vertices in the graph.

Spanning Tree:

A spanning tree is a subset of Graph G, such that all the vertices are connected using minimum possible number of edges. Hence, a spanning tree does not have cycles and a graph may have more than one spanning tree.
